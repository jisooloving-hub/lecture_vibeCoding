> 할 일 관리 앱을 만들고 싶어. PRD를 작성해줘.                                  
매일 10~20개 정도의 할 일을 관리하는 개인용 앱이야.                             
주요 기능은:                                                                    
- 할 일 추가, 수정, 삭제                                                        
- 완료 체크 기능                                                                
- 카테고리 분류 (업무/개인/공부)                                                
- 진행률 보기                                                                   
브라우저에서 바로 실행되고, 새로고침해도 데이터가                               
사라지지 않았으면 좋겠어.                                                       
기술적으로 복잡하지 않게 순수 자바스크립트로 만들고 싶어.                       

● PRD: 개인 할 일 관리 앱

  ---
  1. 개요

  제품명: 개인 할 일 관리 앱
  목적: 하루 10~20개의 할 일을 카테고리별로 관리하고 진행률을 확인할 수 있는
  경량 개인용 웹앱
  타겟 사용자: 개인 사용자 1명
  배포 방식: 브라우저에서 HTML 파일 직접 실행 (서버 불필요)

  ---
  2. 기술 스택

  ┌─────────────┬────────────────────────────────┐
  │    항목     │              선택              │
  ├─────────────┼────────────────────────────────┤
  │ 언어        │ 순수 JavaScript (Vanilla JS)   │
  ├─────────────┼────────────────────────────────┤
  │ 마크업      │ HTML5                          │
  ├─────────────┼────────────────────────────────┤
  │ 스타일      │ CSS3                           │
  ├─────────────┼────────────────────────────────┤
  │ 데이터 저장 │ localStorage                   │
  ├─────────────┼────────────────────────────────┤
  │ 빌드 도구   │ 없음 (단일 파일 또는 3개 파일) │
  └─────────────┴────────────────────────────────┘

  ---
  3. 핵심 기능 요구사항

  3.1 할 일 CRUD

  ┌───────────┬───────────────────────────────────────────────────┐
  │   기능    │                       상세                        │
  ├───────────┼───────────────────────────────────────────────────┤
  │ 추가      │ 텍스트 입력 + 카테고리 선택 후 엔터/버튼으로 추가 │
  ├───────────┼───────────────────────────────────────────────────┤
  │ 수정      │ 항목 클릭 시 인라인 편집                          │
  ├───────────┼───────────────────────────────────────────────────┤
  │ 삭제      │ 삭제 버튼 클릭 (확인 없이 즉시 삭제)              │
  ├───────────┼───────────────────────────────────────────────────┤
  │ 완료 체크 │ 체크박스 클릭 → 취소선 표시, 완료 상태 토글       │
  └───────────┴───────────────────────────────────────────────────┘

  3.2 카테고리 분류

  - 업무 / 개인 / 공부 3가지 고정 카테고리
  - 카테고리별 필터링 (전체 / 업무 / 개인 / 공부)
  - 카테고리별 색상 뱃지로 시각 구분

  3.3 진행률

  - 전체 진행률: 완료 항목 수 / 전체 항목 수 (% + 프로그레스 바)
  - 카테고리별 진행률: 각 카테고리 탭에 완료/전체 표시

  3.4 데이터 유지

  - 모든 데이터는 localStorage에 JSON 저장
  - 새로고침, 브라우저 재시작 후에도 유지
  - 별도 백업/복원 기능 없음 (MVP 범위 외)

● 클로드 코드 단계별 프롬프트

  ---
  STEP 1 — 프로젝트 기반 구조 생성

  HTML/CSS/JS 3개 파일로 할 일 관리 앱의 기본 구조를 만들어줘.

  파일 구조:
  - index.html
  - style.css
  - app.js

  요구사항:
  - index.html에서 style.css, app.js를 연결
  - HTML 기본 레이아웃: 헤더(제목+진행률 바), 카테고리 탭, 입력 영역, 할 일 목록
  - CSS: 깔끔한 카드 스타일, 카테고리별 색상 (업무=파랑, 개인=초록, 공부=주황)
  - app.js: todos 배열과 currentFilter 변수 선언, DOMContentLoaded 이벤트 리스너
  - 외부 라이브러리, CDN 사용 금지 — 순수 HTML/CSS/JS만 사용

  ---
  STEP 2 — localStorage 데이터 레이어 구현

  app.js에 localStorage 기반 데이터 저장/불러오기 함수를 구현해줘.

  구현할 함수:
  - loadTodos(): localStorage에서 'todos' 키로 JSON 파싱해서 반환, 없으면 빈
  배열
  - saveTodos(todos): todos 배열을 JSON.stringify해서 localStorage에 저장
  - generateId(): Date.now() + Math.random() 조합으로 고유 ID 생성

  데이터 구조 (항목 하나):
  {
    id: string,
    text: string,
    category: "업무" | "개인" | "공부",
    done: boolean,
    createdAt: number
  }

  - 앱 시작 시 loadTodos()로 todos 초기화
  - 모든 데이터 변경 후 반드시 saveTodos() 호출

  ---
  STEP 3 — 할 일 CRUD 기능 구현

  app.js에 할 일 추가/수정/삭제/완료 토글 기능을 구현해줘.

  구현할 함수:
  - addTodo(): 입력값과 선택된 카테고리로 새 항목 생성 후 todos에 추가
  - deleteTodo(id): id로 항목 찾아서 todos에서 제거
  - toggleDone(id): 해당 항목의 done 값을 반전
  - editTodo(id, newText): 해당 항목의 text를 업데이트

  UI 동작:
  - 추가: 텍스트 입력 후 엔터 또는 [+] 버튼 클릭
  - 삭제: [✕] 버튼 클릭 시 즉시 삭제 (확인 대화상자 없음)
  - 완료: 체크박스 클릭 시 텍스트에 취소선 표시
  - 수정: 텍스트 더블클릭 시 input으로 전환, 엔터 또는 포커스 아웃 시 저장

  각 동작 후 renderTodos() 호출해서 화면 갱신

  ---
  STEP 4 — 렌더링 및 카테고리 필터 구현

  app.js에 할 일 목록 렌더링과 카테고리 필터 기능을 구현해줘.

  renderTodos():
  - currentFilter 값에 따라 todos 필터링 ('전체' | '업무' | '개인' | '공부')
  - 완료 항목은 목록 하단으로 정렬
  - 각 항목을 HTML로 생성: 체크박스 + 카테고리 뱃지 + 텍스트 + 삭제 버튼
  - 완료 항목에는 CSS class 'done' 추가 (취소선 스타일)

  카테고리 필터 탭:
  - [전체] [업무] [개인] [공부] 탭 클릭 시 currentFilter 변경 후 renderTodos()
  - 활성 탭에 'active' class 추가

  renderProgress():
  - 전체 진행률: 완료 수 / 전체 수 * 100 (%)
  - 프로그레스 바 width와 텍스트 업데이트 (예: "3 / 10 완료 (30%)")
  - renderTodos() 호출 시 renderProgress()도 함께 호출

  ---
  STEP 5 — 마무리 및 UX 개선

  앱의 UX를 개선하고 엣지 케이스를 처리해줘.

  처리할 항목:
  1. 입력 유효성: 빈 문자열이나 공백만 있을 때 추가 차단
  2. 빈 상태 메시지: 할 일이 없을 때 "할 일이 없어요 🎉" 표시
  3. 전체 완료 시: 진행률 바 색상을 초록으로 변경
  4. 카테고리 탭에 항목 수 뱃지 표시 (예: 업무 3)
  5. 추가 후 입력 필드 자동 초기화 및 포커스 이동
  6. 버튼/체크박스 hover 상태 CSS 추가

  최종 확인:
  - 새로고침 후 데이터 유지 여부 확인
  - 항목 추가→완료→삭제 전체 플로우 테스트
  - index.html을 브라우저에서 직접 열었을 때 정상 동작 확인

  toggleDarkMode() — 125번 줄

  function toggleDarkMode() {
    const isDark = document.body.classList.toggle('dark');
    document.getElementById('darkToggle').textContent = isDark ? '☀️' : '🌙';
    localStorage.setItem('darkMode', isDark);
  }

  역할: 버튼 클릭 시 테마 전환
  classList.toggle('dark')로 dark 클래스를 붙이거나 제거하고, 그
  결과(true/false)에 따라 버튼 아이콘과 localStorage 값을 동시에 업데이트한다.
