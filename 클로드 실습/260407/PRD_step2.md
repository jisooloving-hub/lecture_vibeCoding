# PRD Step 2 - 레시피 생성

## 목표
Step 1에서 추출한 재료 목록을 바탕으로 AI가 만들 수 있는 레시피를 추천한다.

## 사용 모델
- `qwen/qwen3.6-plus:free` (OpenRouter API)

## 기능 요구사항

### 레시피 생성
- Step 1에서 전달받은 재료 목록을 API에 전송한다.
- 레시피는 최대 3개까지 생성한다.
- 각 레시피는 다음 정보를 포함한다.
  - 요리 이름
  - 필요 재료 (보유 재료 / 추가 필요 재료 구분)
  - 조리 순서 (단계별)
  - 예상 조리 시간
  - 난이도 (쉬움 / 보통 / 어려움)

### 필터 옵션 (선택)
- 사용자는 레시피 생성 전 조건을 선택할 수 있다.
  - 조리 시간: 30분 이하 / 1시간 이하 / 제한 없음
  - 난이도: 쉬움만 / 전체

### 레시피 카드 표시
- 생성된 레시피를 카드 형태로 나열한다.
- 각 카드에는 요리 이름, 난이도, 예상 시간, 핵심 재료가 표시된다.
- 카드 클릭 시 상세 조리 순서를 펼쳐 보여준다.

### 저장
- 각 레시피 카드에 "저장" 버튼을 제공한다.
- 저장 버튼 클릭 시 Step 3 (사용자 프로필)에 레시피가 저장된다.
- 로그인하지 않은 경우 Step 3 로그인 페이지로 이동한다.

## 화면 구성
```
[인식된 재료 목록 요약]
[필터 옵션]
    ↓ 레시피 생성 중...
[레시피 카드 1]  [저장 버튼]
[레시피 카드 2]  [저장 버튼]
[레시피 카드 3]  [저장 버튼]
[다시 생성하기 버튼]
```

## API 연동 스펙

### 요청
```json
{
  "model": "qwen/qwen3.6-plus:free",
  "messages": [{
    "role": "user",
    "content": "다음 재료로 만들 수 있는 레시피 3가지를 추천해줘: {재료 목록}\n\n각 레시피는 아래 형식으로 작성해:\n**요리명**\n- 난이도: \n- 조리시간: \n- 필요재료: \n- 추가필요재료: \n- 조리순서:\n  1. \n  2. "
  }]
}
```

### 응답 처리
- 마크다운 형식으로 파싱하여 레시피 카드 데이터 구조로 변환
- 파싱 실패 시 원문 텍스트 표시

## 완료 조건
- 재료 목록 전달 후 20초 이내에 레시피 3개가 표시된다.
- 레시피 상세 조리 순서가 단계별로 표시된다.
- 저장 버튼이 정상 동작하여 Step 3에 레시피가 전달된다.
