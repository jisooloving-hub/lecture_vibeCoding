# PRD Step 1 - 냉장고 이미지 인식

## 목표
사용자가 냉장고 사진을 업로드하면, AI가 이미지를 분석하여 식재료 목록을 추출한다.

## 사용 모델
- `google/gemma-3-4b-it:free` (OpenRouter API)

## 기능 요구사항

### 이미지 업로드
- 사용자는 파일 선택 또는 드래그&드롭으로 이미지를 업로드할 수 있다.
- 지원 형식: JPG, PNG, WEBP
- 최대 파일 크기: 10MB
- 업로드 전 이미지 미리보기를 제공한다.

### 재료 인식
- 업로드된 이미지를 OpenRouter API(`google/gemma-3-4b-it:free`)에 전송한다.
- 모델에게 전달할 프롬프트: 냉장고 안에 보이는 식재료를 한국어로 목록화해줘. 각 재료명만 쉼표로 구분해서 나열해.
- API 응답에서 재료 목록을 파싱하여 태그 형태로 화면에 표시한다.

### 재료 수정
- 인식된 재료 목록을 사용자가 직접 추가/삭제할 수 있다.
- 수정된 최종 재료 목록이 Step 2로 전달된다.

## 화면 구성
```
[이미지 업로드 영역]
    ↓ 업로드 후
[냉장고 이미지 미리보기]
[인식된 재료 태그 목록]  ← 추가/삭제 가능
[레시피 추천받기 버튼]  → Step 2로 이동
```

## API 연동 스펙

### 요청
```json
{
  "model": "google/gemma-3-4b-it:free",
  "messages": [{
    "role": "user",
    "content": [
      { "type": "image_url", "image_url": { "url": "<base64 또는 URL>" } },
      { "type": "text", "text": "냉장고 안에 보이는 식재료를 한국어로 목록화해줘. 각 재료명만 쉼표로 구분해서 나열해." }
    ]
  }]
}
```

### 응답 처리
- `choices[0].message.content`에서 쉼표 구분 재료 목록 파싱
- 파싱 실패 시 원문 텍스트 그대로 표시

## 완료 조건
- 이미지 업로드 후 10초 이내에 재료 목록이 표시된다.
- 재료 추가/삭제가 정상 동작한다.
- "레시피 추천받기" 버튼 클릭 시 재료 목록이 Step 2로 전달된다.
