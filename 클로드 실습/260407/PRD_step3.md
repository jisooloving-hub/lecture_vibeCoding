# PRD Step 3 - 사용자 프로필 및 레시피 저장

## 목표
사용자 계정을 관리하고, Step 2에서 마음에 든 레시피를 저장하고 다시 볼 수 있게 한다.

## 기능 요구사항

### 회원 인증
- 이메일 + 비밀번호로 회원가입 / 로그인한다.
- 로그인 상태는 세션(또는 로컬스토리지 토큰)으로 유지한다.
- 비로그인 상태에서도 Step 1~2는 사용 가능하며, 저장 시점에만 로그인을 요구한다.

### 사용자 프로필
- 닉네임, 이메일을 표시한다.
- 식이 제한 설정을 저장한다. (예: 채식, 글루텐 프리, 유제품 제외 등)
  - 설정된 식이 제한은 Step 2 레시피 생성 시 프롬프트에 자동 반영된다.

### 레시피 보관함
- 저장된 레시피 목록을 카드 형태로 표시한다.
- 각 카드에는 요리 이름, 저장 날짜, 난이도, 조리 시간이 표시된다.
- 카드 클릭 시 레시피 상세 내용(조리 순서 포함)을 볼 수 있다.
- 레시피를 삭제할 수 있다.
- 레시피에 메모를 추가할 수 있다. (예: "소금을 줄이니 더 맛있었다")

### 데이터 저장 방식
- 레시피 데이터는 브라우저 로컬스토리지에 저장한다. (백엔드 없이 동작)
- 저장 데이터 구조:
```json
{
  "user": {
    "nickname": "string",
    "email": "string",
    "dietaryRestrictions": ["string"]
  },
  "savedRecipes": [
    {
      "id": "string",
      "name": "string",
      "difficulty": "string",
      "cookingTime": "string",
      "ingredients": ["string"],
      "steps": ["string"],
      "savedAt": "ISO8601",
      "memo": "string"
    }
  ]
}
```

## 화면 구성
```
[프로필 헤더: 닉네임 / 이메일]
[식이 제한 설정]
─────────────────────
[저장된 레시피 목록]
  [레시피 카드]  [메모] [삭제]
  [레시피 카드]  [메모] [삭제]
  ...
[냉장고 다시 분석하기 버튼]  → Step 1로 이동
```

## 완료 조건
- 회원가입 / 로그인 / 로그아웃이 정상 동작한다.
- Step 2에서 저장한 레시피가 보관함에 표시된다.
- 식이 제한 설정이 저장되고, Step 2 레시피 생성 프롬프트에 반영된다.
- 레시피 삭제 및 메모 추가가 정상 동작한다.
