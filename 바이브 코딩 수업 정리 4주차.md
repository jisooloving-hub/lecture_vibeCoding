## 단계별 프롬프트 전략

PRD를 기반으로 프로젝트를 여러 단계로 나누어 AI에게 구체적인 목표를 제시하는 방식.

**기획과 구현의 분리**
- 기획 (클로드 웹): PRD 작성
- 구현 (클로드 코드): 뼈대 → 기능 → UI → 고급 → 최적화

**왜 단계별로 나눠야 할까?**
- 한 번에 모든 기능을 요청하면 품질이 떨어짐
- 각 단계의 결과를 확인한 뒤 다음 단계 진행 가능
- 이전 단계의 결과물 위에 점진적으로 기능 추가

---

## PRD 작성하기

클로드 웹(claude.ai)에서 5W1H 프레임워크로 작성.

```
할 일 관리 앱을 만들고 싶어. PRD를 작성해줘.
매일 10~20개 정도의 할 일을 관리하는 개인용 앱이야.
주요 기능은:
- 할 일 추가, 수정, 삭제
- 완료 체크 기능
- 카테고리 분류 (업무/개인/공부)
- 진행률 보기
브라우저에서 바로 실행되고, 새로고침해도 데이터가 사라지지 않았으면 좋겠어.
기술적으로 복잡하지 않게 순수 자바스크립트로 만들고 싶어.
```

## PRD를 단계별 프롬프트로 변환하기

```
이 PRD를 클로드 코드에서 사용할 단계별 프롬프트로 변환해 줘.
5개의 핵심 단계로 정리해서 각 단계마다 명확한 지시 사항을 만들어 줘.
```

**클로드가 제안하는 5단계**

| 단계 | 내용 |
| --- | --- |
| Step 1 | 기본 구조 및 핵심 기능 구현 |
| Step 2 | 카테고리 기능 및 UI 개선 |
| Step 3 | 진행률 대시보드 추가 |
| Step 4 | 다크 모드 및 고급 기능 |
| Step 5 | 최종 완성 및 최적화 |

## 단계별 프롬프트 실행하기

**클로드 코드 실행 준비**
1. VibeCoding 폴더에 새 폴더 `Study-02` 생성
2. `Study-02` 폴더에서 터미널 열기 → `claude` 실행
3. `/init` 입력하여 프로젝트 초기 설정 (CLAUDE.md 자동 생성)

**Step 1 프롬프트**

```
Step 1: 기본 구조 및 핵심 기능 구현
할 일 관리 앱의 기본 구조를 만들어줘.
요구사항:
1. index.html, style.css, script.js 세 개의 파일로 구성
2. HTML 구조: 제목 "My Tasks", 할 일 입력창과 추가 버튼, 할 일 목록 표시할 컨테이너
3. JavaScript 기능: 할 일 추가, 삭제, 체크박스로 완료 표시, localStorage에 데이터 저장
```

## Step 1 실행 결과 & Step 2

`index.html`, `style.css`, `script.js` 3개 파일 생성됨.

**확인 방법**
- `index.html` 파일을 더블 클릭하여 브라우저에서 실행
- 할 일 입력 → [추가] 버튼 클릭 → 목록에 추가되는지 확인
- 체크박스 클릭, 삭제 버튼 등 기능 테스트

**Step 2 프롬프트**

```
Step 2: 카테고리 기능 및 UI 개선
기존 코드에 카테고리 기능을 추가하고 UI를 개선해줘.
요구사항:
1. 카테고리 기능: 업무(work), 개인(personal), 공부(study)
2. 카테고리별 색상 태그, 필터 버튼
3. 데이터 구조에 category 필드 추가
```

## Step 3: 진행률 대시보드 추가

- 전체 진행률 표시 (“5/10 완료 (50%)” 형식 + 프로그레스 바)
- 카테고리별 미니 진행률, 오늘 추가된 할 일 개수
- 인라인 수정 기능 (더블클릭으로 편집 모드)

---

## Step 4~5: 다크 모드 & 최종 완성

**Step 4: 다크 모드 및 고급 기능**
- 다크/라이트 모드 토글 스위치 + localStorage에 테마 저장
- “완료된 항목 모두 삭제” 버튼
- 키보드 단축키 (`Alt+N`: 새 할 일, `Alt+1,2,3,4`: 카테고리 필터)

**Step 5: 최종 완성 및 최적화**
- 데이터 내보내기/가져오기 (JSON 형식)
- 정렬 옵션 (생성일순, 카테고리순 등)
- 100개 이상 항목에서도 부드러운 성능

## 세션(Session)이란?

클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태.

- 터미널을 닫으면 이전 대화가 사라진 것처럼 보이지만, 실제로는 모든 대화 내용이 프로젝트 폴더에 자동 저장됨
- 며칠 후 이어서 작업해야 할 때 세션 재개가 필수

---

## 세션 재개하기

**방법 1: `claude --resume`**

```
claude --resume
```

이전 대화 기록을 불러와서 마지막 상태 그대로 이어서 작업 가능.

**방법 2: `/resume` 명령어 (별칭: `/continue`)**
- `claude` 실행 후 `/resume` 입력 → 이전 대화를 선택하여 재개

**방법 3: `--continue` 또는 `-c` 옵션**

```
claude --continue
```

가장 최근의 마지막 대화를 자동으로 복구.

## 자동 승인 모드

`Shift + Tab` 키를 누르면 자동 승인 모드로 전환.

- 파일 수정 승인을 매번 누르지 않아도 자동으로 처리
- 반복적인 승인 요청을 생략하여 작업 속도 향상
- 다시 `Shift + Tab`을 누르면 일반 모드로 복귀
- 확인 표시: `>> accept edits on (shift+tab to cycle)`

> ⚠️ 편리하지만 코드 변경 사항을 나중에 반드시 점검할 것!
>

## 컨텍스트란?

현재 대화와 작업 정보를 저장 및 표시한 **메모리 공간**.

- 대화가 길어지면 컨텍스트가 부족해져 자동 압축(auto-compact) 경고 발생

**`/context` — 컨텍스트 상태 확인**

```
/context
```

현재 컨텍스트 사용량을 컬러 그리드로 시각화.

---

## 컨텍스트 압축하기

**`/compact` — 대화 기록 요약**

```
/compact
```

- 대화 기록을 요약하여 컨텍스트 공간을 확보
- 압축 후에도 CLAUDE.md와 프로젝트 파일은 유지됨
- 작업 단위가 끝날 때마다 `/compact`로 정리하는 습관 권장

## 캡처 이미지로 UI 개선하기

멀티모달 AI는 텍스트뿐만 아니라 이미지도 함께 이해하고 처리 가능.

**이미지 전달 방법**
1. Windows 캡처 도구(Snipping Tool) 실행
2. 할 일 관리 앱 화면을 캡처하여 이미지 저장
3. 저장한 이미지를 클로드 코드 명령 프롬프트 창에 드래그 & 드롭
4. 프롬프트 입력창에 `[Image #1]`이 표시됨

**프롬프트 예시**

```
[Image #1] 현재 디자인은 모바일에 최적화되어 있는 것 같아.
데스크톱 환경에서도 전체 화면에서 UI를 볼 수 있도록
디자인을 수정해 줘. 수정한 디자인은 'web_version'이라는
새로운 폴더를 생성한 뒤, 그곳에 저장해 줘.
```

## @ 기호로 파일·폴더 분석하기

`@` 기호를 사용하면 파일이나 폴더를 즉시 참조하고 분석 가능.

**파일 분석 예시**

```
@script.js 파일에서 다크 모드 관련 함수를 찾고 설명해 줘.
```

→ 1,000줄이 넘는 코드에서 특정 함수를 빠르게 찾아 분석

**폴더 분석 예시**

```
@web_version 폴더에 있는 파일들을 분석해서 할 일 관리 앱이
데스크톱 환경에 최적화되었는지 확인해 줘.
```

→ 폴더 내 모든 파일을 종합적으로 분석하여 평가

## 핵심 키워드 정리

### 이론

| 용어 | 설명 |
| --- | --- |
| 단계별 프롬프트 전략 | PRD를 기반으로 프로젝트를 여러 단계로 나누어 구체적인 목표를 제시하는 프롬프트 설계 방식 |
| PRD | Product Requirements Document. 제품의 목적·기능·기술 스펙을 체계적으로 정리한 문서 |
| 세션(Session) | 클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태 |
| 컨텍스트(Context) | 현재 대화와 작업 정보를 저장 및 표시한 메모리 공간 |
| 멀티모달 AI | 텍스트뿐만 아니라 이미지, 음성 등 서로 다른 형태의 데이터를 함께 인식할 수 있는 AI |

### 실습 명령어 ①

| 명령어 | 설명 |
| --- | --- |
| `--resume` | 이전 대화 목록에서 원하는 세션을 선택해 복원 |
| `--continue` (`-c`) | 가장 최근 대화를 자동으로 복구하는 플래그 |
| 자동 승인 모드 | `Shift + Tab`으로 전환, 반복적인 권한 승인을 생략하는 모드 |
| `/compact` | 대화 기록을 요약해 컨텍스트 공간을 확보 |

### 실습 명령어 ②

| 명령어 | 설명 |
| --- | --- |
| `@` 기호 | 파일 및 폴더를 즉시 참조하고 분석할 수 있는 기호 |
| `/config` | 프로젝트 환경 설정을 확인하고 관리하는 명령어 |
| `/doctor` | 시스템 상태를 진단하여 버전, 설치 방식, 업데이트 등을 점검 |
| `/export` | 작업 기록을 클립보드나 파일에 저장하고 문서화 |
