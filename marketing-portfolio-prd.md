# 마케팅 포트폴리오 웹페이지 PRD
**Product Requirements Document v1.0**
작성일: 2026.03.10

---

## 1. 개요 (Overview)

### 배경 및 목적
프리랜서 전환을 준비하는 콘텐츠/SNS 마케터가 채용 담당자 및 잠재 클라이언트에게 **30초 안에 핵심 역량과 수치 성과를 전달**하는 포트폴리오 웹페이지를 구축한다.

### 목표
- 채용 담당자가 랜딩 후 30초 내에 핵심 역량 파악 가능
- 수치 기반 성과(ROI, ROAS, 팔로워 성장률 등)를 시각적으로 임팩트 있게 전달
- 경쟁 지원자 대비 전문성과 신뢰도를 차별화

### 대상 사용자
| 구분 | 설명 |
|------|------|
| 1차 사용자 | 채용 담당자 / 클라이언트 (30초 내 판단) |
| 2차 사용자 | 협업 제안을 고려하는 파트너, 에이전시 |

## 2. 핵심 사용자 시나리오

**시나리오 A — 채용 담당자**
> 링크드인에서 링크를 클릭 → 히어로 섹션에서 핵심 수치 3개 확인 → 대표 캠페인 1~2개 스크롤 → 연락처 클릭

**시나리오 B — 프리랜서 클라이언트**
> 구글 검색 또는 지인 추천 → 서비스 범위 확인 → 성과 수치 신뢰 → 문의 폼 제출

## 3. 페이지 구성 및 기능 요구사항

### 3-1. Hero 섹션 (최우선)
- **30초 법칙 적용**: 이름, 포지션, 핵심 가치 제안(Value Proposition) 한 줄 문구 배치
- **임팩트 숫자 카드 3개**: 대표 성과 수치를 애니메이션 카운터로 표시
  - 예) `ROAS 340%`, `팔로워 +12만`, `CTR 8.2%`
- CTA 버튼: "프로젝트 보기" / "연락하기" 2개

### 3-2. 성과 요약 바 (Stats Bar)
- 스크롤 시 고정되거나 Hero 하단에 위치
- 주요 KPI 4~5개를 한 줄로 나열 (숫자 + 라벨)
- 배경색 또는 구분선으로 시각적 강조

### 3-3. 프로젝트/캠페인 갤러리
- 카드형 레이아웃 (3열 그리드 또는 매거진형)
- 각 카드 필수 요소:
  - 프로젝트 썸네일 이미지
  - 클라이언트/브랜드명 (또는 익명 처리)
  - 역할 태그 (예: `콘텐츠 기획`, `SNS 운영`, `광고 집행`)
  - 핵심 성과 수치 1~2개 (굵게 강조)
- 클릭 시 상세 케이스 스터디 페이지 또는 모달

### 3-4. 케이스 스터디 상세
- 구조: `배경 → 문제 정의 → 전략 → 실행 → 성과`
- 성과 섹션에는 Before/After 비교 차트 또는 수치 강조 블록
- 이미지 갤러리 (콘텐츠 시안, 광고 소재 등)

### 3-5. 스킬 & 툴 섹션
- 마케팅 스킬: 태그 클라우드 또는 카테고리별 리스트
  - 콘텐츠 기획, SNS 운영, 카피라이팅, 광고 운용, 데이터 분석
- 사용 툴: 로고 아이콘 나열 (Meta Ads, GA4, Notion, Figma 등)

### 3-6. About / 나의 이야기
- 간결한 소개 (3~5줄)
- 프리랜서 전환 이유와 방향성 — 신뢰감 형성
- 프로필 사진

### 3-7. 연락처 / CTA 섹션
- 이메일, 링크드인, 인스타그램 링크
- 간단한 문의 폼 (이름, 이메일, 메시지)
- 다운로드 가능한 이력서/소개서 PDF 버튼

## 6. UX/디자인 가이드라인

- **레이아웃**: 여백 넉넉하게, 정보 밀도 낮게 — 채용 담당자의 피로감 최소화
- **컬러**: 신뢰감을 주는 다크 모드 또는 모노크롬 + 포인트 컬러 1개
- **타이포그래피**: 숫자는 크고 굵게 (Display font), 본문은 가독성 우선
- **애니메이션**: 숫자 카운터 애니메이션, 스크롤 페이드인 — 과하지 않게
- **이미지**: 실제 작업물 스크린샷 또는 목업 이미지 사용
