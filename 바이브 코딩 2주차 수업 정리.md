# 02-1 AI를 깨우는 프롬프트의 비밀

### [프롬프트]

: 프롬프트는 내비게이션에 목적지를 입력하는 것과 비슷하다. 

“서울역에서 강남역까지 가는 가장 빠른 길” >> “좋은 데로 데려가 줘”

: AI를 활용한 작업에서 가장 중요한 출발점이 된다.

**좋은 프롬프트의 조건**

- 명확성 : 모호한 표현을 피하고, 원하는 결과를 구체적으로 전달.
- 구체성 : 결과물에 필요한 기능이나 조건을 세부적으로 지시.
- 맥락 제공 : 왜 필요한지, 어떤 상황에서 활용되는지를 알림.

**모호한 프롬프트의 한계**

ex) “멋진 나만의 시작 홈페이지를 만들어 줘.”

→ ‘멋지다’는 표현이 지나치게 모호함 

→ 구체적인 요구사항(세련된 디자인, 깔끔한 디자인, 강력한 기능 등)이 빠져 있으면, AI가 모든 요소를 임의로 결정할 수 밖에 없고, 사용자의 기대나 취향과 다른 결과물이 나올 가능성이 큼.

### [5W1H=좋은 프롬프트]

**PRD : 제품기획서**

: 바이브 코딩에서 PRD=좋은 프롬프트.

**PRD**

1. 무엇을 만들지 명확희 정의

2. 프로젝트 목적과 배경 제시

3. 요구사항, 구현 방법 구체화

**좋은 프롬프트**

1. 해야 할 일을 구체적으로 지시

2. 왜 필요한지, 어떤 상황인지 설명

3. 필요한 기능과 조건 세부 명시

**체계적인 프롬프트를 만드는 틀 5W1H**

- Why : 왜 만드는가?
- Who : 누구를 위한 것인가?
- What : 무엇을 만들 것인가?
- When : 언제 사용되는가?
- Where : 어디서 사용되는가?
- How : 어떻게 만들 것인가?

: Why로 목적을 명확히 하고, Who로 대상을 정의하며, What으로 기능을 구체화 함. When과 Where로 사용 맥락을 파악하고, How로 구현 방법을 결정한다.

: What과 How가 더욱 강조되어야 함.

**효과적인 프롬프트를 위한 기본 질문 템플릿**

<aside>
👩🏻‍💻

“[무엇]을 만들려고 해.

주요 기능은 [구체적인 기능 설명]이야.

[누구]를 위한 것이고,

[왜/어떤 문제를 해결]하기 위해 필요해.

어떻게 구현하면 좋을지 기술적인 방향과 함께 구체적인 PRD를 작성해 줘.”

### 4단계 포트폴리오 전략

| 단계 | 목표 | 핵심활동 |
| --- | --- | --- |
| 1. 뼈대 | HTML 기본 구조 | 섹션 구성 후 섹션마다 이름(ID)부여 |
| 2. 기능 | 실제 데이터, 동작 추가 | 경력, 성과, 케이스 스터디 |
| 3. 디자인 | 시각적 완성도 | 레퍼런스, 색상, 레이아웃 |
| 4. 점검 | 품질 검증 | PRD 대조, 링크 확인 |

: 각 단계마다 요청 → 생성 → 확인 → 수정의 짧은 사이클을 반복해 변경위험을 줄이고, 구조와 품질을 차근차근 확보.
