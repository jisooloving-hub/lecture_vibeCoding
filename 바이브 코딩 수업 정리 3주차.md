## 클로드 코드란?

터미널(명령 프롬프트)에서 실행되는 AI 코딩 도구.

### 클로드 웹 vs 클로드 코드

| 구분 | 클로드 웹 (claude.ai) | 클로드 코드 (Claude Code) |
| --- | --- | --- |
| 실행 환경 | 웹 브라우저 | 내 컴퓨터의 터미널 |
| 작업 방식 | Artifacts에서 미리보기 | 실제 파일 생성·실행 |
| 활용 범위 | 간단한 코드, 문서, 웹페이지 | 복잡한 프로그램, 프로젝트 관리 |
| 파일 접근 | 제한적 | 내 컴퓨터 파일에 직접 접근 |

---

## 터미널이란?

텍스트 명령어로 컴퓨터를 조작하는 도구.

**Windows에서 터미널 실행하는 법**
- 작업 표시줄 검색창에 `cmd` 입력 → Enter
- 또는 `Windows 키 + R` → `cmd` 입력 → 확인

**기본 터미널 명령어**

| 명령어 | 기능 | 예시 |
| --- | --- | --- |
| `dir` | 현재 폴더의 파일 목록 | `dir` |
| `cd` | 폴더 이동 | `cd VibeCoding` |
| `mkdir` | 새 폴더 만들기 | `mkdir Study-01` |

## 실습 ② — MNIST 손글씨 인식 프로그램

### OCR과 MNIST

- **OCR (Optical Character Recognition)**: 이미지 속 글자를 컴퓨터가 읽을 수 있는 디지털 텍스트로 변환하는 기술
- **MNIST**: Modified National Institute of Standards and Technology
    - 용도 : 숫자(0~9) 손글씨 인식 학습에 사용
    - 데이터 : 28×28 픽셀 흑백 이미지 7만 장
    - 특징 : 딥러닝 입문자들이 가장 먼저 접하는 표준 데이터셋

### 프로그램 생성 프롬프트

**cmd에서 클로드 코드 실행**

```cpp
claude
```

```
손글씨로 숫자를 입력하면 이것을 인식하는 코드를 만들어서 실행해 줘.
모든 코드와 주석을 영어로 작성해 줘.
```

**클로드 코드가 자동으로 처리하는 것:**
1. `digit_recognition.py` — 메인 프로그램 코드 생성
2. `requirements.txt` — 필요한 패키지 목록 생성
3. 패키지 설치 및 프로그램 실행

### 실행 결과

- 손글씨 그리기 창 — 마우스로 숫자를 그릴 수 있는 캔버스
- Recognize(인식하기) 버튼 — 클릭하면 AI가 숫자를 인식
- 신뢰도 표시 — 인식 결과와 정확도(%)를 함께 출력

### 배치 파일로 독립 실행

매번 클로드 코드를 열지 않아도 더블클릭으로 실행 가능하게 만들기.

```
숫자 인식 프로그램을 윈도우 탐색기에서 클릭해서 실행할 수 있게 해 줘.
```

→ `.bat` 파일 생성됨

> **배치 파일(.bat)**: Windows에서 여러 명령어를 한 번에 실행하는 스크립트

## 실습 ③ — CLAUDE.md

### CLAUDE.md란?

프로젝트의 구조, 규칙, 명령어를 정리한 설정 파일. 클로드 코드가 시작할 때 자동으로 읽어들임. == “프로젝트 매뉴얼”

| 비유 | 설명 |
| --- | --- |
| 사용 설명서 | 프로젝트에 대한 모든 정보를 담은 문서 |
| 프로젝트 지도 | 폴더 구조와 역할을 한눈에 파악 |
| 팀 규칙 문서 | AI가 따라야 할 코딩 규칙 정의 |

### CLAUDE.md의 구성 항목

| 항목 | 내용 |
| --- | --- |
| Project Overview | 프로젝트 목적과 소개 |
| Commands | 자주 쓰는 명령어 모음 |
| Tech Stack | 사용 기술/라이브러리 |
| Architecture | 폴더 구조와 역할 |
| Code Style | 코딩 규칙과 스타일 |
| Development Notes | 개발 참고 사항 |

### /init으로 .md 자동 생성

```
/init
```

1. 프로젝트 폴더의 파일과 구조를 자동 분석
2. 분석 결과를 바탕으로 CLAUDE.md 파일 생성
3. 프로젝트 목적, 구조, 규칙을 문서화

### # 키로 규칙 빠르게 추가하기

대화 중 `#` 키를 누르면 빠른 규칙 추가 모드 활성화.

```
# 앞으로 만드는 모든 파일은 날짜와 시간을 주석으로 표시해 줘.
```

저장 위치 선택:

| 옵션 | 저장 위치 | 범위 |
| --- | --- | --- |
| User memory | `~/.claude/CLAUDE.md` | 모든 프로젝트에 공통 적용 |
| Project memory | `./CLAUDE.md` | 현재 프로젝트에만 적용 |

> `#` 키는 저장 위치를 선택하는 프롬프트를 표시한 후 저장됨 (즉시 자동 저장 아님)
> 

### 계층적 구조

```
VibeCoding/
├── CLAUDE.md              ← 공통 규칙 (전체 프로젝트)
├── desktop_version/
│   ├── CLAUDE.md          ← 데스크톱 버전 전용 규칙
│   └── digit_recognition.py
└── web_version/
    ├── CLAUDE.md          ← 웹 버전 전용 규칙
    └── (웹 관련 파일들)
```

- **루트 CLAUDE.md**: 프로젝트 전체에 적용되는 공통 규칙
- **하위 폴더 CLAUDE.md**: 해당 폴더에만 적용되는 특화 규칙

### 웹 버전 확장 프롬프트

```
손글씨 인식 프로그램을 웹 버전과 데스크톱 버전으로 나누어 개발하고 싶어.
web_version과 desktop_version 폴더를 만들고 각 폴더에 해당하는 CLAUDE.md 파일도 생성해 줘.
```

**클로드 코드가 하는 일**

1. `desktop_version/` 폴더 - 기존 프로그램을 여기로 이동
2. `web-version/` 폴더 - 웹 버전 코드 생성
3. 각 폴더에 전용 CLAUDE.md 생성
4. 루트에 공통 CLAUDE.md 업데이트

## 자주 발생하는 문제 & 해결법

| 문제 | 원인 | 해결 방법 |
| --- | --- | --- |
| `npm`이 인식되지 않음 | Node.js 미설치 또는 PATH 미등록 | Node.js LTS 버전 재설치 |
| `git`이 인식되지 않음 | Git 미설치 | git-scm.com에서 설치 |
| 클로드 코드 로그인 실패 | 계정 인증 오류 | Claude 계정 재로그인 시도 |
| 손글씨 인식 창이 안 뜸 | Python 또는 패키지 미설치 | 클로드에게 “패키지 다시 설치해 줘” 요청 |
| /init 후 CLAUDE.md 없음 | 폴더 권한 또는 경로 문제 | 올바른 프로젝트 폴더에서 재실행 |
