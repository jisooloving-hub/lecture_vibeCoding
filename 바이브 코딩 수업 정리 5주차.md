## 1. 할루시네이션(Hallucination) (p.174)

### 개념

> AI가 사실과 다른 내용을 **진짜처럼** 만들어내는 현상
> 

### 발생 원인

- AI가 학습 데이터의 패턴을 이어 붙여 **확률적으로 그럴듯한 문장**을 생성하기 때문
- 불명확한 정보로 인한 추론 오류

### 대처 방법

- AI가 생성한 정보는 신뢰할 수 있는 문서 또는 공식 사이트와 비교해 **교차 검증**
- AI에게 `"출처 없는 정보는 생성하지 말 것"` 같은 규칙을 입력하여 **사전 방지**

> ⚠️ AI의 답변을 무조건 신뢰하지 말 것! 특히 사실 기반 콘텐츠(퀴즈, 역사, 과학)는 반드시 교차 검증 필요.

## 4. 할루시네이션 해결하기 (p.180~181)

### 실제 사례: “세계에서 가장 긴 강”

- 정답으로 **아마존강**을 선택했지만 → 틀림! 정답은 **나일강**
- 최근 연구에서 아마존강의 수원지를 재측정한 결과, 더 길다는 주장도 제기됨
- → **논란의 여지가 있는 주제는 교차 검증 후 명확한 기준을 세워야 함**

### 대응 전략

- 검증 가이드라인을 마련하고 **CLAUDE.md 파일에 영구적으로 저장**

> 💡 컴퓨팅 사고 연결: 애매한 표현(“가장 긴”, “최초의”)을 발견하고 명확한 기준(전통적 측정 기준)을 추가하는 것이 **추상화 사고력**입니다.

## 5. 교차 검증 가이드라인 (p.182~185)

### `/memory` 명령어로 CLAUDE.md 편집

```
/memory
```

메모리 저장 위치 선택:
1. **Project memory** — 현재 프로젝트 폴더 안에서만 적용 (`./CLAUDE.md`)
2. **User memory** — 모든 프로젝트에 공통 적용 (`~/.claude/CLAUDE.md`)

> 퀴즈 가이드라인은 현재 프로젝트 전용이므로 **1번 선택**
> 

### CLAUDE.md에 추가할 가이드라인

```
퀴즈 문제 교차 검증 가이드라인

모든 문제 작성 시 확인 사항:

1. 정답이 하나뿐인가?
   - 다른 해석 가능 시 조건 명시 (예: 면적 기준, 2024년 기준)

2. 최상급 표현에 기준이 있는가?
   - '가장 큰', '최초의' 등 표현에 측정 기준 명시

3. 시간과 범위가 명확한가?
   - 변할 수 있는 정보는 시점 명시
   - 지리적, 분류적 범위 한정

4. 교차 검증했는가?
   - 의심스러운 정보는 2개 이상 출처 확인
   - 논란 있는 내용은 주류 학설 기준
```

> 💡 형식이 정해져 있지 않으니 자연어로 간결하게 작성. ‘중요’, ‘반드시’ 같은 강조 표현을 추가하면 AI가 더 잘 따릅니다.

## 8. 커스텀 명령어 (Custom Command) (p.191, p.194)

### 개념

> 사용자가 직접 정의해서 쓸 수 있는 **나만의 단축 지시어**
> 

### 필요한 이유

- 문제 형식 통일, 가이드라인 검증, 난이도 균형 확인 등 **반복 작업이 많음**
- 매번 긴 프롬프트를 입력하면 효율이 떨어짐

### 두 종류

| 종류 | 저장 위치 | 범위 |
| --- | --- | --- |
| 전역 명령어 | `~/.claude/commands/` | 모든 프로젝트에서 사용 가능 |
| 프로젝트 명령어 | `.claude/commands/` | 해당 프로젝트에서만 사용 가능 |

> 명령어는 **마크다운(.md) 파일**로 정의됩니다.
>

## 10. 커스텀 명령어 만들기 (p.195~204)

### `/quiz-validate` — 최상급 표현 검증

`.claude/commands/quiz-validate.md` 생성

- 퀴즈 문제 중 ‘가장’, ‘최초’, ‘최대’ 같은 최상급 표현 탐지
- 측정 기준이 명확한지 점검

**인수 확장** — 카테고리 지정 가능

```
/quiz-validate 한국사
```

`$ARGUMENTS`에 입력된 카테고리를 기준으로 해당 카테고리 문제만 검증

카테고리를 지정하지 않으면 전체 문제 검증

---

### `/quiz-range` — 특정 범위 분석

`.claude/commands/quiz-range.md` 생성

```
/quiz-range 1 20
```

- `$1`번부터 `$2`번까지의 문제 검토
- 난이도 분포 확인
- 정답 위치 편향 분석 (예: 정답이 0번에 55% 집중)
- 균등 분산 권장 사항 제시

---

### `/quiz-add` — 퀴즈 추가 자동화

`.claude/commands/quiz-add.md` 생성

```
/quiz-add 한국사 medium
```

- `$1`: 카테고리, `$2`: 난이도
- `quiz-data.js`에 새로운 문제 자동 추가
- 기존 형식과 일관성 유지
- 검증 가이드라인 자동 준수

## 11. 인수(Argument)와 플레이스홀더(Placeholder) (p.198)

| 개념 | 설명 |
| --- | --- |
| **인수(argument)** | 명령어에 전달하는 추가 정보. “무엇을 대상으로 어떻게 할 것인가”를 구체적으로 지정 |
| **플레이스홀더(placeholder)** | `$ARGUMENTS`, `$1`, `$2`처럼 인수가 들어갈 자리를 표시하는 기호. 실행 시 실제 값으로 치환 |

### 사용 예시

```
/quiz-validate 한국사   ← 특정 카테고리만 검증
/quiz-range 1 20        ← 1~20번 문제만 분석
/quiz-add 한국사 medium ← 한국사 보통 난이도 문제 추가
```
