## Part 1. AI 에이전트

### 에이전트란?

**에이전트(Agent)** — '대리인'이라는 뜻으로, 주어진 목표와 규칙에 따라 **스스로 판단하고 작업을 수행**하는 존재.

**AI 서브에이전트** — 클로드 코드에서 특정 작업을 전문적으로 수행하는 독립형 AI 도우미

- 각각 고유한 역할, 권한, 전문 분야를 가짐
- 메인 AI(클로드 코드)가 서브에이전트에게 작업을 위임

### 커스텀 명령어 vs 서브에이전트

| 구분 | 커스텀 명령어 | 서브에이전트 |
| --- | --- | --- |
| 성격 | 정해진 규칙에 따라 실행 | 상황을 판단하여 유연하게 대응 |
| 비유 | 레시피대로 따라하는 요리사 | 재료를 보고 창의적으로 요리하는 셰프 |
| 적합한 작업 | 반복적·정형화된 작업 | 복잡한 판단이 필요한 작업 |

### 서브에이전트 생성 절차

```
/agents → Create new agent → Project(.claude/agents/)
→ Generate with Claude → 프롬프트 입력
→ 도구 권한 설정 → 모델 선택 → 배경색 설정
```

### 기본 3종 서브에이전트

| 에이전트 | 역할 | 권한 | 모델 | 배경색 |
| --- | --- | --- | --- | --- |
| code-bug-analyzer | 코드 리뷰 & 버그 분석 | Read-only | Opus | Blue |
| performance-optimizer | 성능 최적화 | All tools | Inherit | Green |
| ux-design-advisor | UX 디자인 개선 | All tools | Inherit | Orange |

> 💡 코드 리뷰어는 **Read-only 권한**으로 코드를 읽기만 하고 수정은 하지 않음. 분석 전문 에이전트는 읽기 전용이 안전.
> 

### 서브에이전트 협업 — 직렬 vs 병렬

| 처리 방식 | 설명 | 적합한 경우 |
| --- | --- | --- |
| 직렬 처리 | 순서대로 하나씩 실행 | 앞 작업 결과가 뒤 작업에 필요할 때 |
| 병렬 처리 | 여러 작업을 동시에 실행 | 작업 간 의존성이 없을 때 |

클로드 코드가 작업 성격에 따라 직렬/병렬 방식을 **자동으로 선택**함.

## Part 3. MCP (Model Context Protocol)

### MCP란?

**MCP(Model Context Protocol)** — AI 모델이 외부 데이터, 도구, 서비스와 연결할 때 사용하는 **표준화된 통신 규약**.

### API vs MCP

| 구분 | API | MCP |
| --- | --- | --- |
| 비유 | 특정 가게 하나의 대화 창구 | 여러 가게를 한꺼번에 관리하는 표준 규격 |
| 연결 대상 | 특정 서비스 1개 | 다양한 데이터·도구를 통합 관리 |
| 사용자 | 개발자가 직접 코드 작성 | AI가 스스로 도구를 선택하여 활용 |

### 클라이언트-서버 구조

- **클로드 코드** = MCP 클라이언트 (요청하는 쪽)
- **노션, 깃허브, Playwright 등** = MCP 서버 (서비스를 제공하는 쪽)

### 로컬 MCP vs 원격 MCP

| 구분 | 로컬 MCP | 원격 MCP |
| --- | --- | --- |
| 실행 위치 | 내 컴퓨터 안에서 직접 실행 | 인터넷상 외부 서버에서 실행 |
| 인터넷 필요 | 없음 | 반드시 필요 |
| 설정 | 조금 더 복잡 | 간단 (URL만 지정) |
| 속도 | 빠름 (로컬 실행) | 네트워크에 따라 다름 |
| 보안 | 개인정보 보호에 유리 | 인증 절차 필요 |

```bash
# 원격 MCP 명령어
claude mcp add --transport http [이름] [서버 URL]

# 로컬 MCP 명령어
claude mcp add [이름] [옵션] -- [MCP 실행 명령]
```

### 인증(Authentication)

- **원격 MCP** → 인터넷을 통해 통신하므로 인증 절차 **필수** (API 키, OAuth 등)
- **로컬 MCP** → 내 컴퓨터에서 직접 실행하므로 별도 인증 **불필요**

### MCP 설치 명령어 분석 (Sequential Thinking 예시)

```bash
claude mcp add sequential-thinking -s local
-- cmd /c npx -y @modelcontextprotocol/server-sequential-thinking@latest
```

| 부분 | 의미 |
| --- | --- |
| `sequential-thinking` | 클로드 코드에서 이 MCP를 구분하기 위한 이름 |
| `-s local` | 현재 프로젝트에만 적용 (scope 설정) |
| `--` | 클로드 코드 옵션과 MCP 실행 명령의 구분자 |
| `cmd /c` | 윈도우 환경에서 명령 실행 (맥/리눅스에서는 생략) |
| `npx -y` | Node.js 패키지를 자동 설치하여 실행 |
| `@model...@latest` | 패키지명@버전 (최신 버전) |

> ⚠️ 윈도우 파워셸 환경에서는 `cmd /c` 부분을 삭제!
> 

### 이번 주 MCP 서버 목록

| MCP 서버 | 역할 | 유형 |
| --- | --- | --- |
| Notion | 노션 문서 읽기/쓰기 | 원격 |
| Sequential Thinking | 순차적 추론으로 체계적 분석 | 로컬 |

## Part 4. 테스트와 버전 관리

### 테스트

코드가 의도한 대로 실행되는지, 오류가 없는지 확인하는 과정.

Playwright 같은 자동화 도구를 사용하면 AI가 반복적인 테스트를 대신 수행.

### 버전 관리 & Git 명령어

| 단계 | 명령어 | 설명 |
| --- | --- | --- |
| ① 추가(add) | `git add` | 파일을 준비 공간(staging area)에 추가 |
| ② 커밋(commit) | `git commit` | 현재 상태를 로컬 저장소에 저장 |
| ③ 푸시(push) | `git push` | 로컬 → 원격 저장소(깃허브)에 업로드 |

### 배포(Deploy)

내 애플리케이션을 외부 인터넷에 연결된 서버에 올리는 과정.

**Vercel** — 깃허브에 올리면 자동으로 빌드하고 배포해 주는 클라우드 플랫폼.

### 로컬 스토리지 vs 클라우드 스토리지

| 구분 | 로컬 스토리지 | 클라우드 데이터베이스 |
| --- | --- | --- |
| 저장 위치 | 브라우저 내부 | 인터넷 서버 |
| 공유 | 불가능 (같은 브라우저만) | 가능 (어디서나) |
| 데이터 유지 | 캐시 삭제 시 사라짐 | 영구 보관 |

**Supabase** — 클라우드 데이터베이스를 간단히 자동화해 주는 플랫폼. 무료 플랜 제공, API로 데이터 저장/조회 가능.

## 핵심 키워드 정리

### 에이전트 (07-1)

- **에이전트** — 목표와 규칙에 따라 스스로 판단하고 작업을 수행하는 AI 대리 프로그램
- **AI 서브에이전트** — 클로드 코드에서 특정 작업을 전문적으로 수행하도록 만든 독립형 AI 도우미
- **/agents** — 서브에이전트를 생성하거나 관리하는 클로드 코드 전용 명령어
- **코드 리뷰** — 코드의 버그와 규칙 위반을 점검하고 개선점을 제안하는 품질 검토 과정
- **최적화** — 프로그램의 속도, 자원 사용, 효율성을 향상시키는 성능 개선 작업
- **UX 디자인** — 사용자가 쉽고 편리하게 이용할 수 있도록 화면 구성과 인터페이스를 설계하는 과정

### AI 개발팀 (07-2)

- **제품 기획 관리자** — 제품의 목표, 기능, 사용자 요구사항을 정의하고 전체 개발 일정을 관리
- **백엔드 개발자** — 서버, 데이터베이스 설계와 API 연동 등 핵심 로직을 구현하는 기술 담당자
- **프런트엔드 개발자** — 사용자가 보는 화면과 인터페이스를 설계하고 구현하는 UI 개발자
- **품질 보증 엔지니어** — 기능과 성능을 테스트해 버그를 수정하고 서비스의 안정성을 확보
- **AI 통합 전문가** — AI 모델을 연동, 최적화하기 위해 기술 통합을 담당

### MCP (08-1)

- **MCP** — AI 모델이 외부 데이터, 도구, 서비스와 연결할 때 사용하는 표준화된 통신 규약
- **클라이언트-서버 모델** — 클로드 코드(클라이언트)가 MCP 서버에 요청하고 응답받는 구조
- **로컬 MCP** — MCP 중개 프로그램을 내 컴퓨터 안에서 직접 실행하는 방식
- **원격 MCP** — 중개 프로그램이 인터넷상 외부 서버에서 실행하는 방식
- **인증** — 접속하려는 사람이 정말 허가된 사용자인지 확인하는 절차
- **이중 하이픈 `-`** — 명령어와 옵션을 구분할 때 쓰이는 기호

### 테스트·버전 관리·배포 (08-2, 08-3)

- **테스트** — 코드가 의도한 대로 실행되는지, 오류가 없는지 확인하는 과정
- **버전 관리** — 코드의 변경 이력을 기록하고, 필요할 때 이전 상태로 되돌릴 수 있게 함
- **Context7** — 클로드 코드가 항상 최신 문서를 참고하도록 돕는 지식 연결 도구
- **Playwright** — 웹 사이트를 자동으로 테스트하고 UI를 검증하는 브라우저 제어 도구
- **배포** — 내 애플리케이션을 외부 인터넷에 연결된 서버에 올리는 과정
- **Vercel** — 깃허브에 코드를 올리면 자동으로 빌드·배포해 주는 클라우드 플랫폼
- **Supabase** — 데이터베이스 구축을 자동화해 주는 클라우드 데이터베이스 플랫폼
- **테이블** — 데이터를 표 형태로 저장한 구조 (데이터베이스 안의 개별 엑셀 시트와 유사)

## 셀프 체크 문제

### 07-1

1. 커스텀 명령어와 에이전트의 기술적 차이점으로 옳은 것은?
    - ① 커스텀 명령어는 .md 파일로 저장되지만 에이전트는 .json 파일로 저장된다.
    - **② 에이전트는 독립적인 AI 모델을 선택할 수 있지만 커스텀 명령어는 메인 세션의 모델을 공유한다.**
    - ③ 커스텀 명령어가 에이전트보다 더 복잡한 작업 플로우를 처리할 수 있다.
    - ④ 에이전트는 적용을 위해 재시작이 필요하지만 커스텀 명령어는 바로 사용할 수 있다.
2. 에이전트 생성 시 'Inherit from parent' 옵션을 선택했을 때의 동작은?
    - ① 항상 가장 저렴한 모델을 자동 선택한다.
    - **② 메인 세션에서 사용 중인 모델을 그대로 상속받는다.**
    - ③ 부모 에이전트가 사용하는 모델을 복사한다.
    - ④ Claude의 최신 모델로 자동 업그레이드된다.
