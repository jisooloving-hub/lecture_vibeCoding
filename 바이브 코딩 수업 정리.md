# 바이브 코딩 수업 정리

# 01-1 나의 코딩 파트너, AI 어시스턴트

거대 언어 모델(LLM) : 대규모 언어 AI 모델. 사람의 언어를 이해하고 자연스럽게 대화하듯, 프로그래밍 언어의 문법과 패턴, 그리고 개발자들의 코딩 스타일까지 학습하여 뛰어난 코드 생성 능력을 갖춤.

→ 바이브 코딩이라는 새로운 개발 패러다임의 시작점.

바이브 코딩(vibe coding) : 

- AI와 대화를 주고받으며 프로그램을 구현하는 새로운 개발 패러다임.
- AI에게 아이디어를 자연어로 설명하면 AI가 이를 해석해 코드와 실행 방법을 제시하고, 사람이 이를 검토 및 수정하며 점차 완성도를 높여 나가는 과정.

용어 창시자 : 안드레이 카파시.

LLM을 사용한 코딩 방식의 단점 : gpt, gemini 등

- 채팅창에서 생성된 코드를 일일이 복사해서 실제 개발 환경에 붙여 넣어야 했음.
- 웹 기반으로 작동하는 AI채팅 서비스의 특성상 내 컴퓨터 파일에 접근하거나 프로젝트 전체를 통합적으로 관리하는 데 한계가 있었음.

**AI 통합 도구의 등장**

모델 선택형

: 마치 렌터카를 빌리듯, 프로그램에서 여러 LLM을 선택해 사용할 수 있도록 만든 방식. 코드 편집기와 AI가 통합되어 있음.

: 대표적인 도구는 커서, 리플릿, 윈드서프 등.

- 장점 : 여러 언어 모델 중 특성에 맞는 하나를 골라 편리하게 사용할 수 있음.
- 단점 : 모델과 더불어 도구 자체의 비용을 이중으로 부담함.

전용 모델형

: 특정 AI언어 모델을 직접 내 컴퓨터에 설치해서 사용하는 방식으로, 마치 자기 차를 소유하는 것과 비슷.

: 대표적인 도구는 클로드 코드, 구글 제미나이 CLI 등.

- 장점 : AI 모델 제작사의 전용 프로그램을 설치하므로 추가 비용 없이 모델 사용료만 지불, 로컬 파일에 직접 접근할 수 있어 프로젝트 전체를 효과적으로 관리할 수 있음.

# 01-2 AI와 나만의 첫 웹페이지 만들기

[클로드 바이브 코딩 때 쓰는 용어]

- 아티팩트 : 대화창과 별도로 구분되는 패널.
    - 단순 텍스트 의사소통을 넘어 실제 작동하는 결과물을 즉시 확인할 수 있게 해줌.
    - 매 수정마다 이전 버전이 저장되므로 다양한 시도를 부담 없이 해볼 수 있어서 프롬프트를 단계적으로 발전시키는 법을 자연스럽게 익힐 수 있음.
- 프롬프트 : AI에게 전달하는 요청이나 지시 사항.
    - 개발자가 원하는 결과물의 방향과 품질을 결정하는 핵심 자원.
    - 프롬프트가 구체적이고 맥락을 잘 반영할 수록 AI가 의도에 맞는 코드와 아이디어를 생성할 가능성이 높아진다.
- 커스터마이즈 : 생성 결과를 대화로 계속 개선하는 기능.
    - 첫 결과물이 완벽하지 않아도, 원하는 방향으로 조금씩 고쳐가며 더 나은 결과물을 완성함.
