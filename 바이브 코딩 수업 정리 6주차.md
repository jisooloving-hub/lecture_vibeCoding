## API의 개념과 활용 방법 (교재 p.225)

**API(Application Programming Interface)**란 프로그램과 AI 모델을 이어주는 다리.

- 마치 전화로 전문가에게 도움을 요청하듯, 프로그램이 AI 모델에게 사진 분석을 요청하면 AI가 이미지를 해석하고 결과를 돌려줌
- 웹 애플리케이션은 사용자가 업로드할 인터페이스만 제공하고, 핵심 AI 기능은 API가 처리

**활용 흐름 (‘냉장고를 부탁해’ 예시)**

```
사용자 → [냉장고 사진 업로드] → 앱 → [API] → AI 모델 (이미지 인식)
                                                        ↓
사용자 ← [레시피 추천 표시] ← 앱 ← [API] ← AI 모델 (텍스트 생성)
```

> 복잡한 AI 모델을 직접 만들 필요 없이, API를 통해 이미 구축된 고성능 AI 모델을 바로 사용할 수 있다.
>

## AI 모델 선택하기 (교재 p.226)

### AI 모델을 사용하는 두 가지 방법

| 방법 | 설명 | 예시 |
| --- | --- | --- |
| API 통합 플랫폼 | 여러 AI 모델을 한곳에서 비교·선택 | 오픈라우터(OpenRouter) |
| 각 회사 공식 API | 특정 회사 API를 직접 연동 | OpenAI, Anthropic, Google |

### 오픈라우터(OpenRouter)란?

- 여러 AI 모델 API를 한곳에 모아 제공하는 API 통합 플랫폼
- 클로드, GPT, 라마(Llama) 등 다양한 AI 모델을 **하나의 API 키**로 사용 가능
- 무료 모델도 많아 학습 목적으로 충분히 활용 가능
- 초보자에게 추천: 여러 모델 비교, 무료 체험, 한 곳에서 모든 모델 관리

## API 키란? (교재 p.227)

**API 키(API key)** = API를 사용할 때 사용자를 인증하는 비밀 열쇠

- API를 안전하고 공정하게 사용하기 위한 인증과 관리의 핵심 도구
- 서버는 API 키를 통해 누가, 얼마나 요청을 보냈는지 확인

**API 키가 없다면?**

- 누구나 서버에 접근 → 데이터 유출 위험
- 사용량 추적 불가 → 과도한 요청 제한 불가

### API 키 보안 규칙 ⚠️

- API 키를 코드에 직접 **하드코딩하지 말 것**
- `.env` 파일에 저장하고, `.gitignore`에 등록하여 Git에 올리지 않을 것
- 공유 폴더가 아닌 개인 로컬 컴퓨터에만 보관할 것

## 오픈라우터에서 API 키 발급받기 (교재 p.227~231)

### 발급 절차

1. `openrouter.ai` 접속 → [Sign in] 클릭
2. 구글 계정으로 가입 (또는 이메일 가입)
3. 우측 상단 프로필 아이콘 → [Keys] 메뉴 선택
4. [Create API Key] 클릭
5. Name에 `HonGong-Vibe` 입력 → [Create]
6. 생성된 키를 복사하여 안전한 곳에 저장 **(한 번만 표시됨!)**

### `.env` 파일에 API 키 저장

```
OPENROUTER_API_KEY=sk-or-v1-여기에발급받은키붙여넣기
```

> 메모장에서 위 내용 입력 → [다른 이름으로 저장]
> 
> 
> 파일 이름: `.env` / 파일 형식: 모든 파일(`*.*`) / 인코딩: UTF-8

## AI 모델 연결하고 테스트하기 (교재 p.236~241)

### 클로드 코드가 생성하는 파일들

| 파일 | 역할 |
| --- | --- |
| `.gitignore` | `.env` 파일이 Git에 커밋되지 않도록 보호 |
| `.env.example` | 다른 개발자들을 위한 환경 변수 템플릿 |
| `config.js` | 환경 변수를 안전하게 로드하고 검증하는 유틸리티 |
| `example.js` | OpenRouter API를 사용하는 예제 코드 |
| `package.json` | Node.js 프로젝트 설정 파일 |

### `.env`와 `.gitignore` 파일의 이해 (교재 p.239)

**.env 파일**
- 프로그램의 설정값을 저장하는 특별한 파일
- API 키처럼 민감한 정보를 프로그램 코드와 분리하여 관리
- 코드 수정 없이 `.env` 파일만 바꾸면 설정 변경 가능

**.gitignore 파일**
- `.env` 파일이 깃허브에 업로드되지 않도록 보호하는 설정 파일
- 클로드 코드가 API 키 설정 시 자동으로 `.gitignore`에 `.env`를 등록

냉장고 사진을 업로드하면 AI가 재료를 인식하고, 맞춤 레시피를 추천해 주는 앱.

## 단계별 계획 수립하기 (교재 p.244)

### 3단계 PRD 분해 전략

| PRD | 핵심 기능 | AI 모델 | 일정 |
| --- | --- | --- | --- |
| Step 1 | Streamlit 기반 웹 UI + 냉장고 재료 인식 | Gemma 3 27B-it | 5일 |
| Step 2 | 레시피 생성 + 재료 편집 + 필터링 및 검색 | Qwen 3.6 Plus | 7일 |
| Step 3 | 사용자 인증(회원가입/로그인) + 레시피 저장 + 대시보드 | — | 10일 |

> **컴퓨팅 사고 연결**: 전체 앱을 한 번에 만들지 않고 3단계로 분해하여 점진적으로 구현한다. 각 단계는 이전 단계를 기반으로 발전하며, 최종적으로 완성도 높은 웹 애플리케이션이 된다.

## 단계별 구현 내용

### 1단계: 냉장고 이미지에서 재료 인식하기 (교재 p.246~249)

- Streamlit 웹 프레임워크로 웹 인터페이스 구축
- 이미지 업로드 → Gemma 모델이 자동으로 재료 분석
- 재료명, 수량, 상태가 카드 형태로 표시

> **Streamlit**이란 웹 애플리케이션을 쉽게 만들 수 있도록 도와주는 도구 모음. 파이썬 코드만으로 웹 앱을 만들 수 있어 AI 모델 시연에 매우 유용하다.
> 
> 
> 최신 버전: `pip install streamlit` (v1.56.0, 2026.04 기준, Python ≥ 3.10 필요)
> 

### 2단계: 레시피 생성하기 (교재 p.250~251)

- Qwen 3.6 Plus 모델을 연동하여 레시피 생성 기능 추가
- 재료 편집 기능과 레시피 표시 인터페이스 구현
- 사이드바에 [레시피 생성] 메뉴 추가

**레시피에 포함되는 정보**: 요리명, 난이도, 조리시간, 칼로리, 필요한 재료, 조리 순서, 예상 시간

### 3단계: 사용자 프로필에 레시피 저장하기 (교재 p.252~254)

- 로그인/회원가입 시스템 (프로토타입)
- 개인 대시보드: 저장한 레시피, 요리 횟수, 사용 통계
- [저장] 버튼으로 마음에 드는 레시피 보관
- [나의 레시피] 탭에서 저장된 레시피 확인

**로컬 개발 환경의 이해**: 현재 앱은 로컬 환경(`localhost`)에서만 작동. 다른 사람은 접속 불가 → 실제 서비스를 위해서는 배포 과정이 필요.

## 핵심 키워드 정리

### 이론 (교재 p.242)

| 키워드 | 설명 |
| --- | --- |
| **API** | 외부 AI 서비스와 프로그램을 연결하는 다리. 간단한 요청만으로 복잡한 AI 기능을 활용할 수 있게 해 주는 통신 규칙 |
| **API 통합 플랫폼** | 여러 AI 모델을 한 곳에서 비교하고 선택할 수 있는 서비스 (예: 오픈라우터) |
| **API 키** | API 서비스에 접근하기 위해 발급받아 사용자를 인증하는 비밀 열쇠 |
| **.env** | API 키가 저장된 환경 변수 파일로, API를 호출할 때마다 참조됨 |
| **.gitignore** | `.env` 파일이 자동으로 깃허브에 업로드되지 않도록 특정 파일을 제외하여 보호하는 설정 파일 |

### 실습 (교재 p.255)

| 키워드 | 설명 |
| --- | --- |
| **다중 AI 모델 연동** | 이미지 인식, 텍스트 생성 등 목적에 맞게 여러 AI 모델을 활용하는 방식. 단계별 계획에 따라 점진적으로 완성도를 높여야 함 |
| **웹 프레임워크** | 웹 애플리케이션을 쉽게 만들 수 있도록 도와주는 도구 모음. Streamlit은 파이썬 코드만으로 AI 모델 시연용 웹 앱을 빠르게 구축 가능 |

### Claude Code 새 기능

- `/model` — 대화 중 사용 모델 변경 가능
- `/config` — 설정 관리 인터페이스 (권한, 환경 변수, 도구 설정)
- **MCP 서버** — 외부 도구(DB, 파일 시스템, API)를 구조화해 연동
- **플러그인 / 서브에이전트** — 기능 확장 및 역할별 AI 에이전트 구성
- **데스크톱 앱 & 웹 버전** — 터미널 외에 데스크톱, 웹, VS Code에서 사용

> 기존 `.env` 방식은 그대로 호환된다.
